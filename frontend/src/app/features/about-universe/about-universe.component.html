About · Universe discovery

Discovery builds a dynamic universe from Finnhub symbols plus your strategy filters, then writes one document per day to Firestore: universe/{asof_date} with symbols (full history list), active_symbols (the bot scan list), counts / status_counts, and per-symbol status in the symbols subcollection.

  • Requires FINNHUB_API_KEY and Firestore credentials.
  • Scheduled via GitHub Actions: .github/workflows/universe-discovery-daily.yml (weekdays, premarket), with artifact rotation state.
  • Main scan loads active_symbols. An empty scan list does not silently expand to thousands of inactive tickers.
  • Setup scores are rule strength (0–100), not expected profit. Use Signals for actionable BUY ideas.

Active vs inactive

Each run carries over names from the last merge-days snapshots, re-runs the strategy (capped by --revalidate-cap), and tags every symbol with one of:

  • active — BUY at/above --min-confidence, or WAIT at/above --watch-min-confidence (high-setup watch), then inside top-K. Shown as Active · BUY or Active · Watch.
  • inactive_capped — eligible but ranked outside the top-K cap.
  • inactive_below_min — BUY below the BUY setup floor.
  • inactive_wait — WAIT below the watch setup floor.
  • inactive_sell — strategy returned SELL.
  • inactive_stale — not active for too many runs / too long; never blocks a fresh eligible name.
  • inactive_failed — no signal or no price data this run.

Nothing is deleted; the bot only reads the active_symbols slice.

Local discovery

./run.sh discovery --max-calls 400 --limit 500
# Or with all knobs explicit:
PYTHONPATH=./src python scripts/update_universe_finnhub.py \
  --config config.yaml --max-calls 400 --merge-days 7 \
  --top-k 200 --min-confidence 50 --watch-min-confidence 55 \
  --stale-runs 5 --stale-days 14 --revalidate-cap 1000
