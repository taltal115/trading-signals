About · Universe discovery

Discovery builds a dynamic universe from Finnhub symbols plus your strategy filters, then writes one document per day to Firestore: universe/{asof_date} with symbols (full history list), active_symbols / inactive_symbols, active_count / inactive_count, ts_utc, source, and per-symbol status in symbol_details.

  • Requires FINNHUB_API_KEY and Firestore credentials.
  • Scheduled via GitHub Actions: .github/workflows/universe-discovery-daily.yml (weekdays, premarket), with artifact rotation state.
  • Main scan loads active_symbols via read_universe_for_date (today or latest fallback). Legacy snapshots without that field fall back to symbols.

Active vs inactive

Each run carries over names from the last merge-days snapshots, re-runs the strategy on them (capped by --revalidate-cap), and tags every symbol with one of:

  • active — passed strategy this run and survived top-K.
  • inactive_capped — passed strategy but ranked outside the top-K cap.
  • inactive_low_conf — passed strategy but below --min-confidence.
  • inactive_stale — not active for too many runs / too long (--stale-runs, --stale-days).
  • inactive_failed — strategy returned no signal this run (or no provider data).

Nothing is deleted; the bot just reads the active_symbols slice. Streaks and last_active_at are inherited from the previous snapshot.

Local discovery

./run.sh discovery --max-calls 400 --limit 500
# Or with all knobs explicit:
PYTHONPATH=./src python scripts/update_universe_finnhub.py \
  --config config.yaml --max-calls 400 --merge-days 7 \
  --top-k 200 --min-confidence 0 --stale-runs 5 --stale-days 14 \
  --revalidate-cap 1000
